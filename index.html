10 Principios de Nielsen para la usabilidad web

1. Visibilidad del estado del sistema

Los usuarios deben saber en todo momento qué está sucediendo en el sistema y qué acciones se pueden realizar.

Ejemplo

2. Coincidencia entre el modelo mental del usuario y el sistema

El sistema debe usar un lenguaje y conceptos que sean familiares a los usuarios y que coincidan con sus expectativas.

Ejemplo

3. Facilidad de uso

Los usuarios deben poder usar el sistema de manera fácil y eficiente, incluso si son principiantes.

Ejemplo

4. Eficiencia y eficacia del usuario

Los usuarios deben poder completar sus tareas de forma rápida, precisa y sin errores.

Ejemplo

5. Prevención de errores y facilidad de recuperación

El sistema debe evitar que los usuarios cometan errores y, si lo hacen, debe permitirles corregirlos de forma fácil y sencilla.

Ejemplo

6. Consistencia y mantenimiento de la familiaridad

El sistema debe ser consistente en su comportamiento y mantener las expectativas de los usuarios.

Ejemplo

7. Flexibilidad y adaptabilidad

El sistema debe adaptarse a las diferentes necesidades y preferencias de los usuarios.

Ejemplo

8. Eficacia visual y estética

El sistema debe ser agradable a la vista y usar una estética que sea relevante para la tarea que se está realizando.

Ejemplo

9. Reconocimiento, diagnóstico y recuperación de errores

El sistema debe identificar los errores de forma clara y proporcionar información útil para que los usuarios puedan corregirlos.

Ejemplo

10. Diseño estético y minimalista

Los elementos visuales deben ser claros, concisos y relevantes para la tarea que se está realizando.
