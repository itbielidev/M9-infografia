Reconocer antes que recordar

Los usuarios no deben tener que recordar información de una parte del sistema a otra.

Flexibilidad y eficiencia de uso

El sistema debe ser flexible y eficiente para usuarios con diferentes niveles de experiencia.

Diseño estético y minimalista

El sistema debe ser estéticamente agradable y no sobrecargar a los usuarios con información innecesaria.

Ayuda y documentación

El sistema debe proporcionar ayuda y documentación clara y concisa.

Prevención de errores

El sistema debe ser probado con usuarios reales para garantizar que cumpla con sus necesidades.

Visibilidad del estado del sistema

Definición: El sistema debe mantener informados a los usuarios sobre lo que está sucediendo, a través de comentarios apropiados dentro de un tiempo razonable. ¿Cómo medirlo?Esto se puede medir con pruebas de usabilidad, análisis de comentarios de usuarios. A través de eyetracking o software de registro de usuarios. ¿Solución?Si proporcionamos una retroalimentación clara y oportuna sobre el estado del sistema, como mensajes de confirmación, barras de progreso y animaciones de carga. Nos evitamos que el usuario se sienta frustrado y abandonen la página. Ejemplo nemotécnico:Un ejemplo nemotécnico seria una bombilla encendida para indicar que el sistema esta en funcionamiento. Ejemplo realUn ejemplo real seria just-eat cuando haces un pedido y va actualizando el estado hasta su entrega

Relación entre el sistema y el mundo real

El sistema tiene que “hablar” el lenguaje del usuario con palabras o frases que a éste le sean familiares y que pueda reconocer con facilidad.

Control y libertad del usuario

Tenemos que darle al usuario la posibilidad de subsanar el error y no sentirse frustrado por no poder realizar algo.

Consciencia y estándares

Otro punto que tenemos que tener en cuenta es seguir los convenios establecidos para ciertos iconos.

Ayudar a los usuarios a reconocer, principle-definitiondiagnosticar y corregir los errores

Tenemos que intentar que todos los errores que puedan ocurrir en tu web estén expresados en un lenguaje entendible por todos, no por códigos.
