Reconocer antes que recordar

Definición: Los usuarios no deben tener que recordar información de una parte del sistema a otra.
¿Cómo medirlo? Pruebas de usabilidad, análisis de comentarios de usuarios.
¿Con qué herramientas? Herramientas de análisis de usabilidad, encuestas.
Solución: Minimizar la cantidad de información que los usuarios necesitan recordar, proporcionar etiquetas y descripciones claras, utilizar agrupaciones y jerarquías para organizar la información.
Ejemplo nemotécnico: Una lupa.
Ejemplo real:Google muestra sugerencias de búsqueda a medida que el usuario escribe, lo que le ayuda a recordar o reconocer los términos que busca.

Flexibilidad y eficiencia de uso

Definición: El sistema debe ser flexible y eficiente para usuarios con diferentes niveles de experiencia.
¿Cómo medirlo? Pruebas de usabilidad con usuarios de diferentes niveles de experiencia.
¿Con qué herramientas? Herramientas de análisis de usabilidad, software de registro de usuarios.
Solución: Ofrecer atajos de teclado y funciones avanzadas para usuarios expertos, proporcionar ayuda contextual y tutoriales para principiantes, permitir que los usuarios personalicen la interfaz.
Ejemplo nemotécnico: Un engranaje.
Ejemplo real: Microsoft Word ofrece funciones básicas para usuarios principiantes y opciones avanzadas para usuarios expertos.

Diseño estético y minimalista

Definición: El sistema debe ser estéticamente agradable y no sobrecargar a los usuarios con información innecesaria.
¿Cómo medirlo? Pruebas de usabilidad, análisis de comentarios de usuarios.
¿Con qué herramientas? Herramientas de análisis de usabilidad, encuestas.
Solución: Usar un diseño limpio y simple, evitar el uso excesivo de colores y gráficos, priorizar el contenido sobre la decoración.
Ejemplo nemotécnico: Una lupa que enfoca un elemento específico.
Ejemplo real: Un sitio web que utiliza un diseño minimalista con espacios en blanco y una paleta de colores limitada.

Ayuda y documentación

Definición: El sistema debe proporcionar ayuda y documentación clara y concisa.
¿Cómo medirlo? Pruebas de usabilidad, análisis de comentarios de usuarios.
¿Con qué herramientas? Herramientas de análisis de usabilidad, encuestas.
Solución: Proporcionar un sistema de ayuda contextual, ofrecer documentación completa y fácil de entender, crear tutoriales y guías paso a paso.
Ejemplo nemotécnico: Un signo de interrogación.
Ejemplo real: Un software que ofrece un sistema de ayuda integrado con sugerencias contextuales y acceso a una base de conocimientos en línea.

Prevención de errores

Definición: El sistema debe ser probado con usuarios reales para garantizar que cumpla con sus necesidades.
¿Cómo medirlo? Pruebas de usabilidad, análisis de registros de errores.
¿Con qué herramientas? Software de registro de errores, herramientas de análisis de usabilidad.
Solución: Implementar validación de datos robusta, proporcionar mensajes de error claros y específicos, ofrecer opciones para deshacer y rehacer, permitir que los usuarios reporten errores.
Ejemplo nemotécnico: Un escudo con una cruz.
Ejemplo real: Un sitio web de compras en línea que detecta errores de entrada de datos y ofrece sugerencias para corregirlos.

Visibilidad del estado del sistema

Definición: El sistema debe mantener informados a los usuarios sobre lo que está sucediendo, a través de comentarios apropiados dentro de un tiempo razonable.
¿Cómo medirlo? Esto se puede medir con pruebas de usabilidad, análisis de comentarios de usuarios. ¿Con qué herramientas? A través de eyetracking o software de registro de usuarios.
Solución: Si proporcionamos una retroalimentación clara y oportuna sobre el estado del sistema, como mensajes de confirmación, barras de progreso y animaciones de carga. Nos evitamos que el usuario se sienta frustrado y abandonen la página.
Ejemplo nemotécnico: Un ejemplo nemotécnico seria una bombilla encendida para indicar que el sistema esta en funcionamiento.
Ejemplo real: Un ejemplo real seria just-eat cuando haces un pedido y va actualizando el estado hasta su entrega

Relación entre el sistema y el mundo real

Definición: El sistema tiene que “hablar” el lenguaje del usuario con palabras o frases que a éste le sean familiares y que pueda reconocer con facilidad.
¿Cómo medirlo?Pruebas de usabilidad, análisis de comentarios de usuarios.
¿Con qué herramientas? Encuestas, entrevistas a usuarios.
Solución: Usar un lenguaje sencillo y directo, evitar jerga técnica y siglas.
Ejemplo nemotécnico: Un mapa que relaciona el sistema con el mundo real.
Ejemplo real: Un cajero automático que utiliza palabras y frases familiares para las opciones de menú.

Control y libertad del usuario

Definición: Tenemos que darle al usuario la posibilidad de subsanar el error y no sentirse frustrado por no poder realizar algo.
¿Cómo medirlo? Pruebas de usabilidad, análisis de comentarios de usuarios.
¿Con qué herramientas? Encuestas, entrevistas a usuarios.
Solución: Usar un lenguaje sencillo y directo, evitar jerga técnica y siglas.
Ejemplo nemotécnico:Un mapa que relaciona el sistema con el mundo real.
Ejemplo real: Un cajero automático que utiliza palabras y frases familiares para las opciones de menú.

Consciencia y estándares

Definición: Hay que tener en cuenta es seguir los convenios establecidos para ciertos iconos.
¿Cómo medirlo? Auditorías de usabilidad, análisis de comentarios de usuarios.
¿Con qué herramientas? Herramientas de análisis de usabilidad, guías de estilo.
Solución: Usar una terminología consistente, mantener una apariencia y un diseño uniformes, seguir las convenciones de diseño estándar.
Ejemplo nemotécnico: Un escudo.
Ejemplo real: Un formulario web que valida la entrada de datos y proporciona mensajes de error si la información es incorrecta.
